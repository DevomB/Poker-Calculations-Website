# Kelly & jam toys

Full **Kelly** sizing for binary gambles and **Chubukov-style** symmetric jam models (breakeven stack, jam EV, max profitable jam via binary search). Hand-based jam search composes with `exactHuEquityVsRandomHand` and accepts **`CardInput`** (`string[]` or packed `Uint8Array`) for hole and board.

Toy HU models—validate against your stake structure before production push/fold charts.

| Export | Summary |
| --- | --- |
| [`kellyCriterionBinary`](/docs/reference/api/kelly-and-jam/kelly-criterion-binary) | Full Kelly for binary outcome. |
| [`chubukovSymmetricJamBreakevenStack`](/docs/reference/api/kelly-and-jam/chubukov-symmetric-jam-breakeven-stack) | Symmetric jam breakeven stack from dead money. |
| [`chubukovSymmetricJamEv`](/docs/reference/api/kelly-and-jam/chubukov-symmetric-jam-ev) | Symmetric jam toy EV in chips. |
| [`chubukovMaxSymmetricJamStackChipsBinarySearch`](/docs/reference/api/kelly-and-jam/chubukov-max-symmetric-jam-stack-chips-binary-search) | Max jam stack for fixed equity. |
| [`chubukovMaxSymmetricJamStackBinarySearch`](/docs/reference/api/kelly-and-jam/chubukov-max-symmetric-jam-stack-binary-search) | Max jam from exact HU equity vs random. |
| [`chubukovMaxSymmetricJamStackFromHandBinarySearch`](/docs/reference/api/kelly-and-jam/chubukov-max-symmetric-jam-stack-from-hand-binary-search) | Same search; int32 `maxStackChips` in native. |
