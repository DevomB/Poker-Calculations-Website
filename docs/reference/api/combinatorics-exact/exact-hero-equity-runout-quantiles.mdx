# `exactHeroEquityRunoutQuantiles`

Exact **distribution** of hero equity over all turn/river runouts — mean, variance, and p05/p50/p95 — vs a random villain or an optional weighted range.

<div className="apiSignature">{`exactHeroEquityRunoutQuantiles(heroHoleCards: CardInput, boardCards: CardInput, range?: Float64Array | SparseRangeSpec): HeroEquityRunoutQuantilesResult`}</div>

See [Packed card input](/docs/concepts/packed-card-input) and [`SparseRangeSpec`](/docs/reference/types/sparse-range-spec).

## Import

```js
const poker = require('poker-calculations');
```

## ESM (Node)

```js
import { createRequire } from 'module';
const require = createRequire(import.meta.url);
const poker = require('poker-calculations');
```

## When to use

Mean equity on a draw-heavy board hides bad runouts — enumerate all completions and read the **spread** (tail risk, variance) for study or bot thresholds.

## How to use

```js
const poker = require('poker-calculations');

const hero = ['Ah', 'Kh'];
const flop = ['Qh', 'Jh', '2c'];

const q = poker.exactHeroEquityRunoutQuantiles(hero, flop);

console.log('Mean:', q.mean);
console.log('p05 / p50 / p95:', q.p05, q.p50, q.p95);
console.log('Runouts:', q.n);
```

Omit `range` for equity vs a uniform random villain; pass `Float64Array(1326)` or sparse spec for a weighted range (heavier). Board must be **3 or 4** cards.
