# `expectedValueRaise`

Chip EV of raise vs fold (two-outcome model).

<div className="apiSignature"><code>expectedValueRaise(equityWhenCalled: number, potBeforeRaise: number, raiseSize: number, foldEquity: number, potIfCalled: number): number</code></div>

## Import

```js
const poker = require('poker-calculations');
```

ESM (Node):

```js
import { createRequire } from 'module';
const require = createRequire(import.meta.url);
const poker = require('poker-calculations');
```

## When to use

Two-outcome raise model: villain folds (`foldEquity`) or calls and you realize `equityWhenCalled` in `potIfCalled`, paying `raiseSize`.

## How to use

```js
const equityWhenCalled = 0.45;
const potBeforeRaise = 60;
const raiseSize = 45;
const foldEquity = 0.35;
const potIfCalled = potBeforeRaise + 2 * raiseSize;

const ev = poker.expectedValueRaise(
  equityWhenCalled,
  potBeforeRaise,
  raiseSize,
  foldEquity,
  potIfCalled,
);
console.log('raise EV:', ev);
```
