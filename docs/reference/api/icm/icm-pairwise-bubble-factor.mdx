# `icmPairwiseBubbleFactor`

**Pairwise bubble factor**: ICM loss from losing a pot vs ICM gain from winning it, for hero vs one villain (finite differences on [`icmExpectedPayouts`](/docs/reference/api/icm/icm-expected-payouts)).

<div className="apiSignature"><code>icmPairwiseBubbleFactor(stacks: number[], payouts: number[], heroIndex: number, villainIndex: number, potChips: number): number</code></div>

## Import

```js
const poker = require('poker-calculations');
```

ESM (Node):

```js
import { createRequire } from 'module';
const require = createRequire(import.meta.url);
const poker = require('poker-calculations');
```

## When to use

Tournament **pressure** vs chip EV: compare how much ICM equity hero loses when busting/losing `potChips` to villain versus ICM gained when hero wins the same pot. Values above `1` mean ICM cost of losing exceeds chip-EV-normalized gain from winning.

## How to use

```js
const poker = require('poker-calculations');

const stacks = [12000, 8000, 5000];
const payouts = [400, 250, 150];
const heroIndex = 0;
const villainIndex = 1;
const potChips = 2000;

const bf = poker.icmPairwiseBubbleFactor(
  stacks,
  payouts,
  heroIndex,
  villainIndex,
  potChips,
);
console.log(bf);

const icmBefore = poker.icmExpectedPayouts(stacks, payouts);
console.log({ heroIcm: icmBefore[heroIndex], bubbleFactor: bf });
```

**Returns** `number` — loss/gain ratio from stack perturbations; `heroIndex` and `villainIndex` must be valid indices.
