# `flushMadeFlopToRiverExactProbabilityAsync`

Async flush made; optional AbortSignal.

Card lists use `CardInput` (`string[]` or packed `Uint8Array`). See [Card notation](/docs/concepts/card-notation).

Optional trailing `{ signal?: AbortSignal }` via [`AsyncOptions`](/docs/reference/types/async-options).

<div className="apiSignature">{`flushMadeFlopToRiverExactProbabilityAsync(heroHoleCards: CardInput, flopThree: CardInput, knownDead: CardInput, options?: AsyncOptions): Promise<number>`}</div>

## Import

```js
const poker = require('poker-calculations');
```

ESM (Node):

```js
import { createRequire } from 'module';
const require = createRequire(import.meta.url);
const poker = require('poker-calculations');
```

## When to use

Async sibling of [`flushMadeFlopToRiverExactProbability`](/docs/reference/api/monte-carlo/flush-made-flop-to-river-exact-probability). Optional [`AsyncOptions`](/docs/reference/types/async-options) for cancellation.

## How to use

```js
const hole = ['Ah', 'Kh'];
const flop = ['2h', '7d', '9c'];
const dead = [];

const p = await poker.flushMadeFlopToRiverExactProbabilityAsync(hole, flop, dead);

const ac = new AbortController();
const pending = poker.flushMadeFlopToRiverExactProbabilityAsync(hole, flop, dead, {
  signal: ac.signal,
});
// ac.abort(); // rejects with AbortError
```
