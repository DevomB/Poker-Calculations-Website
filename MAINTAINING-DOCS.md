# Maintaining documentation

API pages under `docs/reference/api/` are **hand-written MDX**. Do not add a script that generates doc content from `index.d.ts` or `binding.cpp`.

When the NPM package adds or renames an export:

1. Update `NPM/native/binding.cpp` and `NPM/index.d.ts`.
2. Add or edit the matching `.mdx` under the correct category folder (kebab-case slug from camelCase).
3. Include **Import**, **When to use**, and **How to use** with a realistic snippet.
4. Add the doc id to `sidebars.ts` if the category uses explicit items (autogenerated categories pick up new files automatically).
5. Run from `Website/`:

   ```bash
   pnpm check:docs
   pnpm build
   ```

`check:docs` only verifies that every `binding.cpp` export has a corresponding MDX slug—it does not generate pages.
